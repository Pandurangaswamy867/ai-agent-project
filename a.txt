
 1A. Fix get_snp_performance (line 93-111)

 Replace 3 hardcoded values with real computations:
 - growth_rate: Compare current 7-day transaction volume vs previous 7-day window. Return "+X.X%" or "-X.X%".
 - fulfillment_index: (completed_or_verified_txs / total_txs * 100) with fallback to snp.fulfillment_reliability * 100.
 - settlement_velocity: Average (updated_at - transaction_date) for verified transactions, formatted as "T+Xh" or "T+X Days",
 fallback to snp.settlement_speed mapping.

 Add new fields to the response:
 - capacity, current_load, capacity_pct (from SNP model)
 - rating, type, commission_rate, supported_sectors (from SNP model)
 - avg_feedback, feedback_count (computed from Partnership.feedback_rating)

 1B. Fix get_performance_metrics (line 86)

 Replace hardcoded "1.2 days" settlement_delay with real computation from verified transaction timestamps.

 1C. New endpoint: GET /mse-activity/{mse_id}

 - Roles: mse, nsic, admin (with ownership check)
 - Returns MSE-scoped activity feed: recent transactions, partnership events, notifications
 - Sorted by timestamp, limited to 8 items
 - Solves: MSE users currently see an empty activity panel (existing /recent-activity is admin-only)

 1D. New endpoint: GET /mse-compliance/{mse_id}

 - Roles: mse, nsic, admin (with ownership check)
 - Returns data-driven compliance status: registration status (from MSE.status), document verification (from OCRDocument), subsidy
 claim status (from Claim)
 - Solves: Compliance card is currently hardcoded with static checkmarks

 1E. New endpoint: GET /snp-trend/{snp_id}

 - Roles: snp (with ownership check)
 - Returns 7-day transaction trend (date, volume, count) — mirrors existing MSE sales_trend endpoint
